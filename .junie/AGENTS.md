## 1. Odoo 15 Architecture and Standards
* The agent must master the standard module structure, including the correct configuration of the `__manifest__.py` file for dependencies, data loading order, licenses, and resource declaration in Odoo 15 using `web.assets_backend`.
* It must apply Odoo's standardized naming conventions for XML identifiers, using the `<model_name>_<view_type>_<suffix>` format.
* Mastery of testing and deployment commands is essential, executing tests via the `--test-enable` parameter and correctly managing module versions in the manifest.

## 2. Backend Development and ORM (Python)
* It must strictly follow PEP8 style guidelines, using `PascalCase` for class definitions, `snake_case` for variables or functions, and dot notation for internal model naming.
* Ability to create models from scratch and inherit existing models (e.g., extending `res.partner`), mastering relational field types (`Many2one`, `One2many`, `Many2many`), numeric and geolocation fields (Float type), binary fields for image uploads, and selection fields.
* Skill in implementing complex business logic, which includes automating state transitions, conditionally assigning different auto-numeric sequences (`ir.sequence`), and creating computed fields to obtain metrics such as handling/response times.
* Expert use of built-in exceptions, specifically raising `odoo.exceptions.ValidationError`, to restrict flows and enforce field population during critical system transitions.

## 3. User Interface and Views (XML)
* Clean XML code creation (4-space indentation) to build Form views with status bars (`statusbar`) and Pivot views (dynamic tables) for operational metric analysis.
* Capacity to develop or adapt simplified visual interfaces for operator use on mobile devices.
* Experience implementing interactive FullCalendar-type views with drag-and-drop functionality for resource planning.

## 4. Security and Permissions
* Flawless management of the `ir.model.access.csv` security file, ensuring proper access to all newly created models.
* Design and configuration of hierarchical roles (Operator, Supervisor, Administrator) that correctly limit read, write, create, and delete permissions according to each user's responsibilities.

## 5. Integrations, Reports, and Automation
* **Webhooks and APIs:** Ability to build and expose endpoints (controllers) in Odoo capable of receiving and processing external POST requests, such as geofence alerts generated by the Samsara telemetry platform, to trigger automatic state changes.
* **QWeb Reports:** Fluent development of dynamic PDF templates that embed photographic evidence galleries, service locations, and integrate logos that change depending on the client.
* **Module Interaction (Inventory):** Ability to link the logistics module with the native inventory application, automating the generation of stock moves or delivery slips based on material parameters, as well as verifying available stock.
* **General Automations:** Configuration of templates for sending automatic email notifications upon task assignment, as well as handling and generating unique QR codes.
